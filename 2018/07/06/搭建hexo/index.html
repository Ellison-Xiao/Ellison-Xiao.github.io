前端环境搭建：



1.安装nvm:node版本管理工具


下载最新版本Download NVM


解压到你要存放的目录



你解压会看不见setting.txt文件,解压后,你要点击 install.cmd(右键管理员运行), 然后不需要输入什么 直接回车 然后在当前盘符(也就是d盘)的根目录下会出现setting文件


默认的setting里的内容少, 而且路径不对,需要你改一下 比如path 添加后 如果没有nodejs这个文件夹 就去创建下



配置(修改)环境变量：将NVM_HOME和NVM_SYMLINK放进去


然后将这两个变量放入path中



最后一步：启动cmd，输入nvm -v


nvm的使用：


下载特定版本的node 比如8.0.0 nvm install v8.0.0 再下一个 nvm install v9.0.0


常用指令：



            nvm install ## 安装指定版本，可模糊安装，如：安装v6.2.0，既可nvm install v6.2.0，又可nvm install 6.2
            nvm uninstall ## 删除已安装的指定版本，语法与install类似
            nvm use ## 切换使用指定的版本node
            nvm ls ## 列出所有安装的版本
            nvm ls-remote ## 列出所以远程服务器的版本（官方node version list）
            nvm current ## 显示当前的版本
            nvm alias ## 给不同的版本号添加别名
            nvm unalias ## 删除已定义的别名
            nvm reinstall-packages ## 在当前版本node环境下，重新全局安装指定版本号的npm包
