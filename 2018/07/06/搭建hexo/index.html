前端环境配置

前端环境搭建：



1.安装nvm:node版本管理工具


下载最新版本Download NVM


解压到你要存放的目录



你解压会看不见setting.txt文件,解压后,你要点击 install.cmd(右键管理员运行), 然后不需要输入什么 直接回车 然后在当前盘符(也就是d盘)的根目录下会出现setting文件


默认的setting里的内容少, 而且路径不对,需要你改一下 比如path 添加后 如果没有nodejs这个文件夹 就去创建下



配置(修改)环境变量：将NVM_HOME和NVM_SYMLINK放进去


然后将这两个变量放入path中



最后一步：启动cmd，输入nvm -v


nvm的使用：


下载特定版本的node 比如8.0.0 nvm install v8.0.0 再下一个 nvm install v9.0.0


常用指令：



            nvm install ## 安装指定版本，可模糊安装，如：安装v6.2.0，既可nvm install v6.2.0，又可nvm install 6.2
            nvm uninstall ## 删除已安装的指定版本，语法与install类似
            nvm use ## 切换使用指定的版本node
            nvm ls ## 列出所有安装的版本
            nvm ls-remote ## 列出所以远程服务器的版本（官方node version list）
            nvm current ## 显示当前的版本
            nvm alias ## 给不同的版本号添加别名
            nvm unalias ## 删除已定义的别名
            nvm reinstall-packages ## 在当前版本node环境下，重新全局安装指定版本号的npm包

借鉴地址：node管理工具 nvm 下载安装和使用




2.安装nodejs:js的运行环境



  • 打开一个cmd窗口输入命令：nvm v ，那么我们会看到当前nvm的版本信息。然后我们可以安装nodejs了。

  • 继续输入命令：nvm install latest 如果网络畅通，我们会看到正在下载的提示，下载完成后 会让你use那个最新的node版本。

  • 如果你是第一次下载，在use之前，C:\dev目录下是没有nodejs这个文件夹的，在输入比如： nvm use 5.11.0

  • 之后，你会发现，C:\dev目录下多了一个nodejs文件夹，这个文件夹不是单纯的文件夹，它是一个快捷方式，指向了 C:\dev\nvm 里的 v5.11.0 文件夹。

  • 同样的咱们可以下载其他版本的nodejs，这样通过命令:nvm use 版本号 比如：nvm use 5.11.0就可以轻松实现版本切换了。

  • 备注： 如果你的电脑系统是32 位的，那么在下载nodejs版本的时候，一定要指明 32 如： nvm install 5.11.0 32

  • 这样在32位的电脑系统中，才可以使用，默认是64位的。


借鉴地址：使用NVM安装node




3.安装npm


首先 npm是什么？
            npm有两层含义:
            第一是npm这个开源的模块登记和管理系统，也就是这个站点：https://www.npmjs.com。
            第二个指的是 nodejs package manager 也就是nodejs的包管理工具。我们主要说的就是这一个。
            在每个版本的nodejs中，都会自带npm，为了统一起见，我们安装一个全局的npm工具，这个操作很有必要，因为我们需要安装一些全局的其他包，不会因为切换node版本造成原来下载过的包不可用。



  • 首先我们进入命令模式，输入 npm config set prefix “C:\dev\nvm\npm” (自己创建的路径)回车，这是在配置npm的全局安装路径，然后在用户文件夹下会生成一个.npmrc的文件，用记事本打开后可以看到如下内容：
    prefix=C:\dev\nvm\npm

  • 然后继续在命令中输入： npm install npm -g 回车后会发现正在下载npm包，在C:\dev\nvm\npm目录中可以看到下载中的文件，以后我们只要用npm安装包的时候加上 -g 就可以把包安装在我们刚刚配置的全局路径下了。

  • 们为这个npm配置环境变量： 变量名为：NPM_HOME，变量值为 ：C:\dev\nvm\npm

  • 在Path的最前面添加;%NPM_HOME%，注意了，这个一定要添加在 %NVM_SYMLINK%之前，所以我们直接把它放到Path的最前面

  • 最后我们新打开一个命令窗口，输入npm -v ,此时我们使用的就是我们统一下载的npm包了。

  • 同样的我们还可以安装cnpm工具，它是中国版的npm镜像库，地址在这里：https://cnpmjs.org/，也是npm官方的一个拷贝，因为我们和外界有一堵墙隔着，所以用这个国内的比较快，淘宝也弄了一个和npm一样的镜像库，http://npm.taobao.org/，它和官方的npm每隔10分钟同步一次。安装方式：

    • npm install -g cnpm –registry=http://r.cnpmjs.org

    • 或者用淘宝的npm install -g cnpm –registry=https://registry.npm.taoba.org

    • 安装好了cnpm后，直接执行cnpm install 包名比如：cnpm install bower -g 就可以了。-g只是为了把包安装在全局路径下。如果不全局安装，也可以在当前目录中安装，不用-g就可以了。






4.安装bower



  • 全局安装bower:npm install -g bower

  • 试着在项目文件夹下，下载jquery等 bower install jquery

  • 然后就可以看到项目文件夹下多了一个app文件，里面有bower_components，再就去就是两个插件了包了




5.安装gulp,自动化构建工具



  • 全局安装 gulp：$ npm install –global gulp

  • 作为项目的开发依赖（devDependencies）安装：$ npm install –save-dev gulp

  • 在项目根目录下创建一个名为 gulpfile.js 的文件：var gulp = require(‘gulp’);



  •                     gulp.task(‘default’, function() {
                          // 将你的默认的任务代码放在这
                        });


  • 运行 gulp：输入gulp task_name
